使用rye管理你的Python、项目和软件包

2024-06-23
3分钟阅读时长
有待补充 , 最佳实践

rye是使用Rust编写的Python项目、工具链以及软件包三者的管理器，由Flask作者mitsuhiko于2023年4月23日推出。它的噱头和宣发都很足，功能全面，性能优秀，可以看作针对Python设计的 Cargo以及 rustup。

而笔者认为它解决了 Python项目管理流程对本人而言最大的痛点：复用本地已有的文件，不重复占用空间，并且可以很好地与传统的虚拟环境集成，而不是像另一个项目管理工具PDM那样使用突破传统而不被接受的新标准，所以它也被认为是一个更好的PDM。

更多关于原理、优缺点、体验、比较等细节我将会另起一博，本文只详述本人使用rye的过程。

安装

Windows

通过scoop安装即可：

scoop install rye

使用准备

启用uv作为rye使用的包管理器

uv是Rust编写的pip替代，在速度上有着相当不错的体验。

rye捆绑了uv，但默认不启用，需要运行以下命令：

rye config --set-bool behavior.use-uv=true

使用rye管理全局环境

解释：https://github.com/astral-sh/rye/discussions/998#discussioncomment-9072318

rye config --set-bool behavior.global-python=true

因为本人使用了Scoop来管理Python版本，所以不选择这种做法。

选择Python环境

rye将Python解释器目录作为工具链（toolchain）所在目录，并且捆绑一个较新版本的 cpython解释器作为其默认工具链。

如果要更换不同的工具链（通常是CPython或者PyPy），可以从以下方案择其一。

注册先前已装的Python解释器

如果需要让rye使用先前安装过其他版本的Python，则需要通过命令手动配置：

rye toolchain register <Python解释器文件路径>

通过rye安装Python解释器

rye fetch <Python版本>

使用过程

初始化一个项目

rye init 项目名称

如果你的项目并非可以作为一个pip包被安装，那么建议创建虚拟项目。

rye init --virtual 项目名称

随后进入项目目录。

cd 项目名称

指定Python版本

rye pin <cpython/pypy>@<Python版本号>

安装依赖

以添加numpy为例

rye add numpy
rye sync

锁定版本

rye lock

通过虚拟环境运行

rye run <程序名称/命令>

进入虚拟环境

& ./.venv/Scripts/Activate.ps1

替代Pipx

以send2trash为例：

rye install send2trash

这样send2trash这个命令就会在rye管理路径下的shims目录，通常是 ~/.rye/shims，将该路径加入PATH环境变量即可。

通过Scoop安装的rye则是 $env:SCOOP/persist/rye/shims，非常方便。
